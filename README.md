# 음성 기반 자산·소모품 재고관리 시스템

구글 스프레드시트를 DB로 활용하는 음성 기반 재고관리 웹/앱 프로토타입입니다.

## 아키텍처 개요

```
┌─────────────────────────────────────────────────────────────┐
```

### 기술 스택 선정 이유

| 계층 | 선택 | 이유 |
|------|------|------|
| Frontend | **React + Vite** | 모바일/웹 반응형, 터치 서명 패드, Web Speech API |
| Backend | **Node.js + Express** | `googleapis` 공식 SDK, Nodemailer, node-cron 통합 용이 |
| DB | **Google Sheets API** | PRD 요구사항, 비개발자도 시트에서 직접 수정 가능 |
| STT | **Web Speech API + GPT** | 브라우저 내장 STT(무료) + GPT 구조화 파싱 |
| Storage | **Google Drive** | 서명 PNG 저장 + `=IMAGE()` 시트 삽입 |

## Google Cloud 사전 준비

1. Google Sheets API, Google Drive API 활성화
2. 서비스 계정 JSON → `backend/credentials/service-account.json`
3. 스프레드시트 4탭 생성 후 서비스 계정을 **편집자**로 공유
4. Drive에 서명 저장 폴더 생성 → 폴더 ID를 `.env`에 설정
5. `backend/.env.example`을 `.env`로 복사 후 값 입력

## OpenAI (선택)

`OPENAI_API_KEY` 설정 시 GPT-4o-mini 파싱. 미설정 시 mock 파서 동작.

## 프로젝트 기획서 
Talkment(토크먼트) 최종 제품 요구사항 및 설계 명세서
1. 시스템 핵심 설계 원칙
개념: "음성으로 관리하는 창고 및 재고 관리 플랫폼"
멀티 워크스페이스 구조: 1명의 관리자 계정은 여러개의 프로젝트 (관리 공간)생성 가능
내부적으로 프로젝트 1개당 독립된 구글 스프레드 시트 파일 1개 (3시트 세트) 가 매칭되어 생성
대쉬보드 첫 화면에서 카드형태의 프로젝트 중 선택
각 프로젝트 카드에는 현재 총 재고 종류수, 반납 지연, 물품 수량 부족(최소 재고 수량 데이터 있을 때) 등 간략 표시
하단에 프로젝트 추가 버튼으로 새로운 작업공간 추가 가능
사용자 정의형 아키텍처: 반출 이력 시트에는 고정된 양식을 강제하지 않고, 사용자가 켜고 끈 기능(시리얼, 반납 등)에 따라 시스템과 구글 시트 구조가 동적으로 변함.
반납 안내 메일 발송 : 명단, 발납일이 있는 경우 반납일 1일전~1일후 총 3회 안내 메일 발송. 
인프라: Replit (24시간 구동 클라우드 백엔드 서버) + Google Sheets API (데이터 스토리지).
2. 데이터 베이스 및 구글 시트 구조 (3-Set 구조)
워크스페이스가 생성되면 구글 시트 파일 내에 다음 3개의 탭이 자동 생성/연동됩니다.
① [명단] 시트 (선택 사항)
반출 권한이 있는 사원/사용자 목록
컬럼: 이름, 부서, 직급, 이메일(반납일 1일 전 메일 발송), 연락처
② [물품관리] 시트 (현재 재고 마스터 대시보드)
일반 소모품과 시리얼 관리 품목이 섞여 있는 현재 창고 상태.
컬럼: 품목 ID, 품목명, 시리얼 넘버(있을 때만), 규격/상세, 현재 상태(보유중/반출완료), 재고 수량, 최소 재고 수량, 비고
시리얼 품목 규칙: 품목명이 같아도 시리얼별로 한 줄씩 생성, 초기 수량은 무조건 1 고정.
일반 소모품 규칙: 시리얼 칸은 빈칸, 재고 수량 칸에 총합(예: 50) 기입.
③ [반출이력] 시트 (반출/반입 로그 누적)
기본 필수 컬럼: 일시 (오늘 날짜 자동), 품목명, 반출 수량
동적 활성화 컬럼 (사용자 선택에 따름): 시리얼 넘버, 반출자, 반납 예정일, 반출자 서명
사용자 정의 커스텀 컬럼: 사용자가 추가한 임의의 항목 (예: 프로젝트명, 용도 등)이 우측에 동적 추가됨.

3. 핵심 비즈니스 프로세스 (Data Flow)
① 초기 온보딩 및 세팅 프로세스
사용자가 가입 후 [새 작업 공간 생성] 클릭.
구글 드라이브 내 엑셀 시트 등록.명단(선택), 물품 목록(필수) 시트가 1셋트
(시트 등록 시 시리얼 또는 반납 필요 물품인지, 일반 소모품 출고인지 선택)
액셀 첫행과 필수 값 매핑 진행 
[필수/선택 항목 세팅] 화면에서 사용할 필드(시리얼, 반출자 등)를 체크하고 커스텀 필드를 추가함.
[세팅 완료] 버튼 클릭 시, 시스템이 구글 드라이브에 표준화된 3-Set 시트 파일을 생성하고 데이터 변환 이적 완료.
② 음성 반출/반납 처리 프로세스
사용자가 스마트폰 앱에서 마이크 버튼을 누르고 말함. (예: "마케팅팀 홍길동, 맥북 AP-1234 반출")
백엔드(Node.js) AI 파싱 모듈이 활성화된 필드 스키마를 기준으로 텍스트 분석.
활성화된 필드 매칭: 반출자: 홍길동, 시리얼: AP-1234, 품목명: 맥북.
구글 시트 연동 트랜잭션 실행:
[물품관리] 시트에서 AP-1234 행을 찾아 상태를 [보유중 ➡️ 반출완료], 수량을 [1 ➡️ 0]으로 수정.
[반출이력] 시트 맨 아래에 새로운 행(Row)으로 [오늘 날짜, 맥북, 1, AP-1234, 홍길동]을 추가(Append).
음성 및 화면으로 "맥북 AP-1234번 반출 처리가 완료되었습니다." 피드백 제공.
4. 앱 UI 대시보드 화면 구성
📱 화면 1: [메인 대시보드 (실시간 재고 현황)]
상단: 커다란 [음성인식 말풍선] 버튼 (클릭 시 "말씀하세요..." 애니메이션 파형 작동).
중앙: 재고 목록 리스트뷰 ([물품관리] 시트 실시간 연동).
시리얼 품목인 경우 초록색 라벨(보유중) 또는 빨간색 라벨(반출완료)로 직관적 표시.
일반 소모품은 우측에 대형 숫자로 재고: 45개 표시.
최소수량 데이터 있는 경우 재고 부족 알림 표시
하단: 하단 탭 바 (재고현황 / 반출이력 / 설정).
📱 화면 2: [초기 세팅 & 매핑 화면]
엑셀 업로드 영역: 구글 드라이브 파일 선택 UI (Google Drive Picker)
디자인: 깔끔한 구글 드라이브 아이콘과 함께 [ 내 구글 드라이브에서 파일 선택하기 ] 대형 버튼 배치.
작동: 클릭 시 사용자의 드라이브 폴더/파일 구조가 앱 내 모달(Modal) 창으로 시각화되어 노출되며, 검색 및 선택이 가능함.
관리 유형 선택 : 시리얼 관리 (자산 개별 추적), 수량 관리 (소모품, 재고)
시리얼 선택 시 : “다음 칼럼이 필요합니다.품목명, 시리얼 넘버, 상태”
수량 관리 : 품목명, 재고수량, 최소수량 (선택)
컬럼 매핑 UI: 선택한 구글 시트의 첫 번째 행 헤더 데이터를 기반으로 시스템 필드와 일치시키는 드롭다운 매니저.
동적 필수 필드 활성화 스위치 그룹: 관리자가 장부 컬럼을 켜고 끌 수 있는 체크박스 및 토글 UI (시리얼, 반출자, 반납일 등).
커스텀 추가 폼:[ + 직접 항목 추가 ] 버튼을 통한 커스텀 필드 확장 폼 (최대 10개).
📱 화면 3: [반출이력 장부 화면]
[반출이력] 시트와 실시간 연동되는 타임라인 형태의 영수증 리스트.
"누가, 언제, 무엇을" 가져갔는지 최신순으로 정렬되어 노출.
상단에 검색 바 및 필터(기간별, 인물별) 제공.
